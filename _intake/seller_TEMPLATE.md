# Seller intake — canonical field set

> **This is a field-set reference, not an offline form.** Intake completion happens in chat. Same workflow as `buyer_TEMPLATE.md`.

## How to use

Same pattern as buyer intake: copy the block if you have all the fields, paste raw info if you have some, paste a name and phone if that's all you've got. 01 walks the rest conversationally.

## Field set

```
**Seller lead**

Name:
Phone:
Email:
Lead source: (referral / Zillow / door knock / past client / other)

**Property:**
Address:
Property type: (single-family / condo / townhouse / other)
Currently occupied: (owner / tenants / vacant)

**Timeline:**
Target list date:
Why now (life event, market read, opportunistic):
Already with another agent (Y/N):

**Mortgage situation:**
Paid off / mortgaged:
Approximate equity (if known):

**Price expectation:**
Target sale price (or "no idea yet"):
Recent CMA done: (Y/N — by whom)
Any online estimates seen (Zillow, Redfin):

**Property condition / improvements:**
Year built / square feet (if known):
Recent updates:
Known issues or repairs needed:

**Constraints:**
Tenants in place:
Estate / probate situation:
Anything else 02 should know before pulling listing-side comps:
```

## What 01 needs at minimum to open the case file

Name + intent (selling) + address (or at least ZIP code so 02 knows the territory). Everything else can be classified and pursued via the gap rule.
